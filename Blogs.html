Understanding JavaScript Fundamentals

Get started with the basics of JavaScript

JavaScript (JS) operates as a synchronous, single-threaded language, executing code line by line. Each line of code executes sequentially, and the execution of the second line occurs only after the first one completes.

Core Concepts of JavaScript

Execution Context:

  • All JavaScript code operates within an execution context, comprising two components:
  • Memory or Variable Environment: Stores variables or functions, such as `A: 10` or `function { ... }`.
  • Code or Thread of Execution: Executes the code line by line.

When the variable allocation and code execution phase completes within the execution context, the entire execution context is deleted or destroyed. All execution contexts are stored in the call stack, with each function invocation creating a new execution context within the global execution context.

Working of Functions in JavaScript & Variable Environment

Hoisting in JavaScript refers to the process where variable declarations and function declarations are moved to the top of their scope before code execution. This means that all declarations are processed before the code is executed.

For example:


                        var x=1;
                        a();
                        b();
                        console.log(x);

                        function a(){
                            var x = 10;
                            console.log(x);
                        }
                        function b(){
                            var x = 100;
                            console.log(x);
                        }

Shortest JavaScript Program, window, and this Keywords

In JavaScript, declaring a variable like var x = 10; assigns it the initial value of undefined before execution. This placeholder value is visible in the browser's scope before executing the program. If a variable is referenced before declaration, it results in an undefined output. However, accessing an undeclared variable leads to an error of "not defined." Remember, undefined is not empty; it serves as a placeholder value.

For example:


                    console.log(x); // output undefined
                    var x=10; // here before executing this line it will assign undefined for var x
                    console.log(x);
                    console.log(a); // here we get error as not defined

JavaScript's Loosely Typed Nature

JavaScript is a loosely typed or weakly typed language, allowing variables to be assigned any data type such as boolean, integer, or double. However, it's advisable not to assign undefined to any variable, as it's considered a poor practice.
